# Java面向对象编程实践：一个文字冒险战斗模拟器

## 1. 项目背景

这是我在学习Java语言和面向对象编程（OOP）思想时，完成的一个课程设计项目。

项目的目标是构建一个基于文本的冒险与战斗模拟系统。我希望通过这个项目，将课堂上学到的**继承、封装、多态**等OOP核心概念，以及**工厂模式**等设计模式，应用到一个具体的、逻辑复杂的实践中。

整个项目完全使用Java实现，通过控制台进行输入和输出。

---

## 2. 系统核心设计

我将整个游戏世界抽象成了一系列相互协作的Java类，核心设计如下：

#### **`Adventurer` (冒险者类)**
这是系统的核心类，封装了冒险者的所有属性和行为。
*   **属性:** 姓名、ID、攻击力、防御力、生命值。
*   **物品管理:** 拥有自己的`bottles`(药水)和`equipments`(装备)列表，以及一个`bag`(背包)。
*   **核心行为:**
    *   `drink(bottleId)`: 使用背包中的药水，根据药水类型（`HpBottle`, `AtkBottle`...）恢复生命或提升属性。
    *   `attack(target)`: 对其他冒险者发起攻击，伤害计算会考虑双方的攻防属性。
    *   `employ(adventurer)`: 雇佣其他冒险者，形成一个团队。
    *   `askHelp()`: 当生命值过低时，可以向团队中的其他成员求助。

#### **物品系统 (Item System)**
*   **继承结构:**
    *   设计了一个抽象的`Bottle`基类，并派生出`HpBottle`, `AtkBottle`, `DefBottle`三个子类，体现了**继承**关系。
    *   `Equipment`类则封装了所有装备的通用属性。
*   **背包 (`Bag`) 类:** 每个冒险者都有一个`Bag`实例，负责管理实际携带的物品，并包含了一些业务逻辑，如背包容量限制、同名装备替换等。

#### **战斗与奖励系统 (Combat & Reward System)**
这是展现**多态**和**设计模式**思想的核心部分。
*   **接口设计:**
    *   `Gud` (Guard) 接口：定义了所有守护者的通用行为 `fight(adventurer)`。
    *   `Treasure` 接口：定义了所有宝物的通用行为 `usedBy(adventurer)`。
*   **策略模式 (Strategy Pattern) 的应用:** `Shd`, `Flm`, `Stn` 等类都实现了`Gud`接口，但各自的`fight`逻辑（战斗力判定）不同。这使得在不修改主逻辑的情况下，可以轻松地增加新的守护者。
*   **工厂模式 (Factory Pattern) 的应用:**
    *   我设计了一个`TreasureFactory`类，它的职责是**根据传入的守护者类型，创建并返回对应的宝物实例**。
    *   这种做法将“创建宝物”的逻辑与“使用宝物”的逻辑完全**解耦**，使得主流程 (`Mysterious.java`) 无需关心宝物的具体创建过程，极大地提高了代码的可读性和可维护性。

---

## 3. 技术与能力总结

*   **面向对象编程 (OOP):** 深入实践了封装、继承和多态三大核心概念。
*   **设计模式:** 成功应用了**工厂模式**来解耦对象的创建和使用，并通过接口实现了**策略模式**的基本思想。
*   **Java核心技术:** 熟练使用了Java的集合框架，如`ArrayList`, `HashMap`, `HashSet`等来管理复杂的数据关系。
*   **系统设计能力:** 能够将一个复杂的游戏系统，分解成多个职责单一、相互协作的类，展现了良好的软件工程实践。

## 4. 如何运行

这是一个标准的Java项目，可以通过IDE（如IntelliJ IDEA, Eclipse）或命令行运行。

1.  克隆本仓库到本地。
2.  使用IDE打开项目，它会自动识别`src`目录为源码根目录。
3.  找到 `MainClass.java` 文件，并运行其`main`方法。
4.  程序将通过控制台（Terminal）进行交互，你可以根据提示输入指令。
